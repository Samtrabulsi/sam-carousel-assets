# GHL Social Publisher

Direct REST workflow for scheduling posts into the GoHighLevel Social Planner. Uses a Private Integration Token end-to-end — no MCP wrapper, no third-party file host, multi-tenant by design.

## What it does

1. Renders an HTML file to a PNG at the right size for your target feed (default: IG portrait 1080×1350)
2. Uploads the PNG directly to your GHL location's media library
3. Schedules a post on Facebook / Instagram / LinkedIn / X / TikTok in one call
4. Verifies the post landed in the scheduler

## Install

```bash
cd /path/to/carousel-system/publisher
pip install -r requirements.txt
python -m playwright install chromium     # one-time, ~150MB
```

## First-time onboarding for a new client

See [INSTALL.md](INSTALL.md) for a step-by-step.

Short version:

```bash
# 1. Create a Private Integration Token in GHL (Settings → Private Integrations).
#    See INSTALL.md for the exact scopes needed.

# 2. Add the token to .env:
echo "GHL_PIT_ACME_CO=pit-xxxxxxxx" >> ../.env

# 3. Auto-discover the client's location, accounts, and user:
python -m publisher.cli setup --client acme-co --token pit-xxxxxxxx --location-id <LOC_ID>
# Writes clients/acme-co.toml with everything discovered.

# 4. Schedule a test post (dry-run first):
python -m publisher.cli publish \
  --client acme-co \
  --html path/to/post.html \
  --caption-file path/to/caption.txt \
  --schedule "2026-05-22 10:00 America/New_York" \
  --platforms facebook instagram \
  --dry-run
```

## CLI reference

### `setup` — onboard a new client

```bash
python -m publisher.cli setup \
  --client <slug> \
  --token <PIT> \
  --location-id <LOC_ID> \
  [--user-id <USER_ID>]
```

Hits `/locations/{id}`, `/users/`, and `/social-media-posting/{loc}/accounts` to write `clients/<slug>.toml` pre-filled with location info, the author user_id, and one account per platform (skipping any with expired OAuth tokens).

### `publish` — schedule a post

```bash
python -m publisher.cli publish \
  --client <slug> \
  --html FILE [FILE ...] \              # multiple HTML files = carousel
  --caption "..." | --caption-file FILE \
  --schedule "2026-05-21 10:00 Europe/Athens" \
  [--platforms facebook instagram ...] \  # omit = all configured accounts
  [--size ig_portrait|square|fb_landscape|story] \
  [--post-type post|story|reel] \
  [--dry-run]                              # render only, skip upload+schedule
```

Sizes:
- `ig_portrait` 1080×1350 (default, 4:5)
- `square` 1080×1080
- `fb_landscape` 1200×630
- `story` 1080×1920

### `verify` — confirm or list posts

```bash
# By ID
python -m publisher.cli verify --client <slug> --post-id <POST_ID>

# By window
python -m publisher.cli verify --client <slug> \
  --from-date 2026-05-21T00:00:00Z \
  --to-date   2026-05-31T00:00:00Z \
  --status    scheduled
```

## Layout

```
publisher/
├── __init__.py
├── cli.py              # argparse subcommands
├── ghl_client.py       # REST client (services.leadconnectorhq.com)
├── render.py           # HTML → PNG (Playwright, size presets)
├── config.py           # TOML + .env loading
├── clients/
│   ├── _template.toml
│   └── <slug>.toml     # one per client; generated by `setup`
├── requirements.txt
├── README.md
└── INSTALL.md
```

The PIT lives in `../.env` as `GHL_PIT_<UPPER_SLUG>`. Never in code, never in TOML.

## Adding more clients

Each client gets:
1. A row in `.env`: `GHL_PIT_<SLUG>=<their-PIT>`
2. A file `clients/<slug>.toml` (auto-generated via `setup`, or hand-edited from `_template.toml`)

Then every command targets that client with `--client <slug>`.

## Authoring HTML to match the size preset

The publisher renders at whatever viewport the preset dictates — the HTML must lay out for that size. Mismatches show up as empty space (HTML smaller than preset) or cropped content (HTML larger).

Recommended root rule in your HTML, parameterized to the preset:

```css
html, body, .frame {
  width:  1080px;
  height: 1350px;   /* match the preset: 1350 for ig_portrait, 1080 for square, etc. */
  background: var(--poster-bg);
}
```

If you keep one HTML template for all sizes, drive `width`/`height` from CSS variables and override per render. Or maintain one HTML per preset.

## Known limits

- IG video / Reel uploads aren't tested in v1 — extend `mime_for` in `cli.py` and confirm with GHL docs
- The `verify` listing uses `POST /social-media-posting/statistics/{loc}` — if your GHL build uses a different read endpoint, adjust `GHLClient.list_posts`
- Expired OAuth tokens (LinkedIn/X/TikTok happen often): `setup` flags them and comments them out in the TOML — reconnect each in GHL UI then re-run setup
